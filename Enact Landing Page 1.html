How it works

1

Pick your systems

Start with first-class support for HubSpot, Salesforce, and Postgres. Enact ships canonical actions like create_contact, update_deal, insert_row.

2

Define your policies

Use a simple Python DSL to define global and user-based policies. No LLMs. Just deterministic functions versioned in Git.

3

Define workflows

Describe the workflows your agent is allowed to run: new_lead_workflow, enrich_and_route, convert_to_opportunity.

4

Call Enact

Your agent calls a single function. Enact validates state, runs policies, executes allowed actions, and returns both the result and a human-readable receipt.

Ship agents in days, not weeks

from enact import EnactClient, workflows, policies

enact = EnactClient(
    systems=[
        HubSpot(api_key="..."),
        Salesforce(api_key="..."),
        Postgres(dsn="..."),
    ],
    policies=[
        policies.no_duplicate_contacts(),
        policies.contractor_cannot_write_pii(),
        policies.limit_tasks_per_contact(max_tasks=3, window_days=7),
    ],
    workflows=[
        workflows.new_lead_workflow,
        workflows.convert_lead_workflow,
    ],
)

# Your agent calls this
result, receipt = enact.run(
    workflow="new_lead_workflow",
    actor_email="agent@company.com",
    payload={
        "email": "jane@acme.com",
        "company": "Acme Inc",
        "source": "outbound",
    },
)

# Result: contact created, deal created, receipt generated
