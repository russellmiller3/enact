Quickstart

Running in 5 minutes

Install, connect your systems, define policies, call enact.run(). That's it.

1

Install

One package. No infra required to start.

2

Connect your systems

HubSpot, Salesforce, Postgres ‚Äî pass your credentials, get canonical actions back.

3

Define policies

Deterministic Python functions. No LLMs. Versioned in Git. Testable with pytest.

4

Call enact.run()

Your agent gets a result and a receipt. Blocked actions are logged with a reason.

# Step 1 ‚Äî install  (from enact.cloud, not the other enact)
pip install enact-sdk

# Step 2‚Äì4 ‚Äî in your agent code
from enact import EnactClient, workflows, policies

enact = EnactClient(
    systems=[
        HubSpot(api_key="..."),
        Postgres(dsn="postgresql://..."),
    ],
    policies=[
        policies.no_duplicate_contacts(),
        policies.contractor_cannot_write_pii(),
        policies.limit_tasks_per_contact(max_tasks=3, window_days=7),
    ],
    workflows=[workflows.new_lead_workflow],
)

# Your agent calls this one function
result, receipt = enact.run(
    workflow="new_lead_workflow",
    actor_email="agent@company.com",
    payload={"email": "jane@acme.com", "company": "Acme Inc"},
)

# Works with any framework that can call a Python function

Using LangChain, Claude tool_use, or OpenAI? Wrap it in a tool:

from langchain_core.tools import tool

@tool("enact_new_lead")
def enact_new_lead(payload: dict) -> dict:
    """Safely run the new_lead_workflow via Enact"""
    result, _ = enact.run(workflow="new_lead_workflow", actor_email="agent@company.com", payload=payload)
    return result

# Same pattern works with Claude tool_use, OpenAI Responses API, CrewAI, AutoGen, etc.

Workflow library

Stop writing boilerplate connector code

Think Zapier ‚Äî but for AI agents, built by engineers who've seen what goes wrong. Every workflow is pre-built, red-teamed, idempotent, and ready to deploy in minutes. We write them. We maintain them. You just register and run.

No more figuring out connector quirks, handling rate limits, or writing the same duplicate-check logic for the fifth time. The library grows with every release.

Why pre-built workflows matter
‚úì

Idempotent by design. If step 1 succeeds and step 2 fails, a retry won't create duplicates. Every connector checks before it acts.

‚úì

Red-teamed. We run adversarial prompts against every workflow before it ships. If an agent can break it, we fix it first.

‚úì

Policy-ready. Every workflow ships with a recommended policy set. You can tighten them. You can't accidentally skip them.

Roadmap

What's coming next

v0.2+

These are the capabilities teams ask for most. Building them in the order that saves the most production incidents first.

Human-in-Loop Gates coming soon

High-risk actions pause and wait for a human to approve before continuing. Slack DM, email, or web UI ‚Äî your choice. Timeout = auto-abort with receipt. No more "the agent just did it while I was asleep."

One-Step Rollback coming soon

Every receipt-logged action can be undone in one command. enact.rollback(receipt_id) reverses what the agent did, generates a rollback receipt, and notifies on-call. Your undo button for agent mistakes.

Vertical Policy Packs coming soon

Pre-built policy sets for your industry ‚Äî FinTech (no trading outside market hours, PII in transit controls), Healthcare (HIPAA field protection, PHI access audit), DevOps (prod deploy gates, blast-radius limits). Install, tune, done.

Compliance Export coming soon

Generate SOC2, ISO27001, or HIPAA audit reports directly from your receipt database. Every agent action already logged and signed ‚Äî turn it into a compliance artifact in one click. Hand it to your auditor, not your engineers.

Anomaly Detection coming soon

Learn your agents' normal behavior. Flag deviations automatically ‚Äî an agent touching 10x its usual number of records, a new action it's never called before, activity at unusual hours. Catch the subtle stuff before it becomes a postmortem.

Multi-Agent Arbitration coming soon

When multiple agents want to modify the same resource simultaneously, Enact serializes and arbitrates ‚Äî not your database. Define merge strategies, conflict policies, and priority rules. Your agents stop stepping on each other.
