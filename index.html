How it works

Every Bash command flows through a policy engine first.

Claude Code emits a PreToolUse hook on every tool call. Enact Code is a tiny binary on that hook. Synthesizes the command into a structured payload, runs it through deterministic Python policies, returns deny JSON if any policy blocks. Sub-50ms overhead. No LLMs in the decision path.

The non-obvious finding: Claude's training already self-refuses 11 of 15 dangerous prompts — modern models are surprisingly careful. But 4 still slip through and cause damage: DROP TABLE customers on a one-line "we don't need it" prompt, DELETE FROM users, aws s3 rm --recursive on a "staging" bucket, and the rename-then-drop adversarial bypass. Enact's policy engine catches all four, deterministically. Self-refusal drifts with model versions; code doesn't.

Try it yourself

30 seconds. Two commands. In any repo.

No account, no signup, no API key. The hook runs locally and writes signed receipts to ./receipts/. You own the policies. You own the audit trail. We can't see any of it.

  1. 1. Install the SDK pip install enact-sdk — adds the enact-code-hook binary to your PATH.
  2. 2. Initialize in your repo From your project root: enact-code-hook init. Writes .claude/settings.json (merge-safe — preserves your other hooks), creates .enact/policies.py with sensible defaults, generates a 32-byte HMAC secret, gitignores the config dir.
  3. 3. Open Claude Code in the repo That's it. Every Bash call now runs through Enact's policy engine. Try asking Claude to drop a table, force-push, or commit your .env — watch it get blocked with a clear reason.
  4. 4. Customize Edit .enact/policies.py. Add your own protected tables, your own forbidden patterns, your own time-of-day restrictions. Reloads on every command. 
# What gets blocked by default

from enact.policies.git import (
    dont_force_push,
    dont_commit_api_keys,
)
from enact.policies.db import (
    protect_tables, block_ddl,
)
from enact.policies.time import (
    code_freeze_active,
)

POLICIES = [
    code_freeze_active,
    block_ddl,             # DROP / TRUNCATE
    dont_force_push,       # --force / -f
    dont_commit_api_keys,  # sk-… / AKIA / ghp_…
    protect_tables([
        "users", "customers",
        "orders", "payments",
        "audit_log",
    ]),
]
